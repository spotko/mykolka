# Mykolka's Game

Стартовий каркас для веб-версії "Миколкиної гри".

## Що вже є

- `Next.js + TypeScript + Tailwind` каркас
- стартова сторінка з базовим стилем
- структура, від якої ми підемо до реальної онлайн-гри

## Що потрібно встановити перед запуском

На цьому комп'ютері поки немає `Node.js`, тому спочатку його треба поставити.

Найпростіший шлях для macOS:

1. Встановити `nvm` з офіційної інструкції.
2. Встановити Node LTS:

```bash
nvm install --lts
nvm use --lts
```

3. Перевірити:

```bash
node -v
npm -v
```

## Як запустити проєкт

```bash
npm install
npm run dev
```

Потім відкрити [http://localhost:3000](http://localhost:3000).

### Опційно: вмикання локального persistence для кімнат

За замовчуванням кімнати живуть лише в пам'яті процеса.

Якщо хочете, щоб вони переживали перезапуск dev-сервера локально:

```bash
ROOM_STORAGE=file npm run dev
```

Тоді кімнати зберігатимуться у `.data/rooms.json`.

### Підготовка під Redis

У проєкті вже є `Redis`-адаптер через REST env-змінні. Коли будете готові:

```bash
ROOM_STORAGE=redis \
REDIS_REST_URL=... \
REDIS_REST_TOKEN=... \
npm run dev
```

## Найближчі кроки

1. Додати чисту логіку раунду: хід, відкриття карти, завершення раунду.
2. Реалізувати батл і визначення програвшого.
3. Підключити `Socket.IO` для онлайн-гри.
4. Перенести реальні дії гравців на сервер і синхронізувати кімнату.

## Демо-сторінки

- `http://localhost:3000/` - головна сторінка
- `http://localhost:3000/room/demo` - демо кімнати з раундом, чергою ходу і штрафами

## Онлайн-кімнати

1. На головній сторінці введіть ім'я і створіть кімнату.
2. Скопіюйте код кімнати другу.
3. Друг відкриває головну сторінку, вводить код і заходить у ту ж кімнату.
4. Поки що синхронізація зроблена через серверний стан і періодичне оновлення, без Socket.IO.

## Зберігання кімнат

Зараз кімнати йдуть через абстракцію сховища:

- `server/room-store.ts` - правила роботи з кімнатами
- `server/room-repository.ts` - інтерфейс сховища
- поточні адаптери: `in-memory`, `file`, `redis`

Це вже підготовано під наступний крок: підключення `Redis` без переписування логіки гри.
