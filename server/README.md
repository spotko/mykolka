# Server notes

Поточна архітектура:

- `server/room-store.ts` містить правила роботи з кімнатами
- `server/room-repository.ts` дає абстракцію сховища кімнат
- зараз активний `in-memory` адаптер

Що це нам дає:

- ігрова логіка більше не залежить напряму від `Map`
- наступним кроком ми можемо додати `Redis` без великого рефакторингу

Найближчий крок:

1. Додати `RedisRoomRepository`
2. Вибирати адаптер через env, наприклад `ROOM_STORAGE=memory|redis`
3. Зберігати кімнати в Redis з TTL
4. За потреби перенести realtime-події на Redis pub/sub
